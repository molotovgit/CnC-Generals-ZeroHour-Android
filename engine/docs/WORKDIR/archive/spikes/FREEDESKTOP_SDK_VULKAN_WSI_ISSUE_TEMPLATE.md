# Freedesktop SDK Issue Template - Vulkan WSI / XCB Symbol Gap

## Title

Vulkan ICD load failure in Flatpak runtime 25.08 due to missing symbol `xcb_dri3_import_syncobj_checked`

## Summary

A Vulkan application running inside Flatpak runtime `org.freedesktop.Platform//25.08` fails at ICD load time with:

```
undefined symbol: xcb_dri3_import_syncobj_checked
```

The same application works on native host (outside Flatpak) using host Vulkan stack.

## Environment

- Runtime: `org.freedesktop.Platform//25.08`
- SDK: `org.freedesktop.Sdk//25.08`
- Host distro: Ubuntu 24.04 (also expected to affect other distros when sandbox uses same runtime)
- GPU stack: Mesa Vulkan drivers

## Reproduction

1. Install runtime and app.
2. Run app with Vulkan loader debug enabled:

```bash
flatpak run --command=bash com.fbraz3.GeneralsXZH -c 'export VK_LOADER_DEBUG=all; /app/bin/GeneralsXZH -win -noshellmap'
```

3. Observe Vulkan loader failure when loading ICD library from GL provider path.

## Observed Output

```text
[Vulkan Loader] ERROR: /usr/lib/x86_64-linux-gnu/GL/default/lib/libvulkan_radeon.so: undefined symbol: xcb_dri3_import_syncobj_checked
[Vulkan Loader] ERROR | DRIVER: loader_icd_scan: Failed loading library associated with ICD JSON
```

## Additional Evidence

Inside sandbox:

```bash
nm /app/lib/libxcb-dri3.so.0.1.0 | grep xcb_dri3_import_syncobj_checked
```

Result:

```text
Symbol not found
```

## Expected Behavior

Vulkan ICD should load without unresolved symbol errors when using runtime-provided XCB stack.

## Request

Please confirm whether runtime `25.08` should ship a newer `libxcb-dri3` (or related package set) compatible with current Mesa Vulkan ICD requirements, and advise recommended fix path.
